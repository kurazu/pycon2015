Threads and GIL

The Python interpreter is not fully thread-safe. In order to support multi-threaded Python programs, thereâ€™s a global lock, called the global interpreter lock or GIL, that must be held by the current thread before it can safely access Python objects.

Threads and GIL

Therefore, the rule exists that only the thread that has acquired the GIL may operate on Python objects or call Python/C API functions.

Threads and GIL

The lock is also released around potentially blocking I/O operations like reading or writing a file, so that other Python threads can run in the meantime.
